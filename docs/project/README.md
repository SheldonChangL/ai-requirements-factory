# Project Notes

This directory contains maintainer-facing project-tracking documents that evolve with development.

These files are useful for contributors who want roadmap context, but they are not part of the stable public API or setup surface.

Feature-level specs and implementation plans now live alongside this directory under `docs/specs/` and `docs/plans/`. Use `docs/project/` for roadmap and status tracking, and use the spec/plan templates for concrete feature execution.

- `project_status.md`: current implementation status and latest completed work
- `milestones.md`: formal milestone checkpoints and the next phase plan
- `todo_tasks.md`: active follow-up tasks and workflow priorities
- `approved-artifact-knowledge-writeback-plan.md`: proposal for turning approved stage outputs into reusable product knowledge
- `TASKS.md`: completed open-source preparation checklist from the initial cleanup phase
- `REFACTOR_CHECKLIST.md`: completed workflow refactor checklist for the modular delivery architecture
