# Open Source vs Hosted Boundary

## Open source scope

The open source project keeps these capabilities in-repo:

- Requirement interview workflow
- Prompt profiles
- Model adapter registry
- File ingestion
- PRD, architecture, and user story generation
- Markdown and JSON export
- Jira push workflow

## Explicitly not in the open source MVP

- Team workspaces
- SSO
- Role-based access control
- Approval workflows
- Audit logs
- Usage analytics
- Managed hosting and support operations

## Hosted / commercial direction

If a hosted version is built later, its value should come from governance and team operations, not from closing the workflow engine itself.

That means hosted-only value can include:

- SSO and organization management
- Fine-grained permissions
- Review and approval flows
- Audit history
- Usage dashboards
- Managed infrastructure and support
